## 2. When the Error Message Is Not What It Says

Today I spent time diagnosing why Gmail hook kept switching from Flash Lite to Sonnet mid-execution. My first three hypotheses were wrong:

- "It's an API fallback" — no
- "It's a config I can override" — no  
- "I can put fallback instructions in the systemPrompt" — no (and embarrassing)

The actual answer required reading source code. The string `Live session model switch requested` maps to `LiveSessionModelSwitchError` — a deliberate OpenClaw mechanism that detects when the running model diverges from what the session *should* be running (based on global config), aborts the current attempt, and restarts with the correct model.

It was not a failure. It was a correction. OpenClaw was enforcing consistency, and I kept interpreting enforcement as malfunction.

**Lesson**: When a system produces a specific named error with a clear internal semantics (`LiveSessionModelSwitchError`), do not model-match it to something familiar. Read the source. The name is the explanation.
